Submit an **order lifecycle event** into the Fulfillment service before or after key milestones (e.g., order confirmed). To submit an order event, send a <span class="inline-flex items-center rounded-md bg-blue-50 px-2 py-1 text-xs font-medium text-blue-700 inset-ring inset-ring-blue-700/10">POST</span> request to `/v1/Events/OrderEvent`

<br>

:ApiEndpointTryIt{
  method="POST"
  base-url="https://logistics-fulfillment-api.test.eshopworld.net"
  path="/v1/Events/OrderEvent"
  defaults='{
    "headers": { "Content-Type": "application/json" },
    "body": {
      "id": "4f1b8a1e-1111-4c22-9a9a-aaaaaaa00001",
      "tenantCode": "ACME",
      "eswOrderReference": "ESW-ORDER-123456",
      "orderType": "Sale",
      "orderConfirmed": true,
      "status": "Created",
      "deliveryCountryIso": "IE",
      "creationDateTime": "2025-10-29T10:00:00Z"
    }
  }'
}[Try it]


### Required Parameters  
 
The OrderEvent schema does **not** mark specific fields as required, but the following are typically expected for a meaningful event

| Parameters            | Type       | Description |
|----------------------|------------|-------------|
| id                   | GUID       | Unique event identifier. |
| tenantCode           | String     | ESW tenant code associated with the order. |
| status               | String     | Current order status represented by this event (e.g., `OrderConfirmed`). |
| creationDateTime     | DateTime   | Event creation timestamp (ISO 8601, UTC). |
| lastUpdateDateTime   | DateTime   | Event last update timestamp (ISO 8601, UTC). |


### Optional Parameters

The following parameters are supported by the endpoint and may be sent when available: 

| Optional Parameters        | Type                | Description |
|---------------------------|---------------------|-------------|
| monolithId                 | Int64               | Internal order id. |
| orderType                  | String              | Order category/type. |
| brandOrderReference        | String              | Retailer’s order reference. |
| parentBrandOrderReference  | String              | Parent order reference (if nested). |
| eswOrderReference          | String              | ESW order reference. |
| orderConfirmed             | Boolean             | Indicates order confirmation state. |
| deliveryCountryIso         | String              | 2-letter ISO of delivery country. |
| currencies[]               | Array\<Currency\>   | Currency breakdowns. |
| fxRates[]                  | Array\<FxRate\>     | FX rates in effect for the order. |
| retailerInvoice            | RetailerInvoice     | Invoice number/date from retailer. |
| aggregateIds[]             | Array\<AggregateId\>| Correlation identifiers. |
| shopperExperience          | ShopperExperience   | Shopper experience metadata. |
| metadataItems[]            | Array               | Extra metadata entries. |


What happens if your request is accepted and processed:
- The service returns **200 – OK** to indicate the event was accepted. 
- Downstream systems can consume the event to update fulfillment and reporting workflows.

::field-group

  ::field{name="tenantCode" type="string" required}
  ESW tenant code for the request. Non-empty string.
  ::

  ::field{name="fulfillmentCentreId" type="string" required}
  Identifier of the fulfillment center where inventory is held.
  ::

  ::field{name="provider" type="string" required}
  Name of the inventory source (e.g., 3PL or WMS).
  ::

  ::field{name="items" type="array" required}
  Array of inventory item objects to upsert for this fulfillment center.
  ::

  ::field{name="items[].sku" type="string" required}
  Retailer SKU used on the storefront. Length 3–32 characters.
  ::

  ::field{name="items[].articleStatus" type="string" required}
  Current status of the article.  
  **Enum** `"Expected"`, `"Active"`, `"Quarantine"`, `"Suspended"`.
  ::

  ::field{name="items[].quantity" type="integer"}
  Quantity on hand for the SKU at this fulfillment center.
  ::

  ::field{name="items[].variantProductCode" type="string"}
  Master/variant product code that groups related SKUs.
  ::

  ::field{name="items[].upc" type="string"}
  UPC barcode for the SKU, if available.
  ::

  ::field{name="items[].ean" type="string"}
  EAN barcode for the SKU, if available.
  ::

  ::field{name="items[].size" type="string"}
  Variant size (e.g., `L`, `EU 42`).
  ::

  ::field{name="items[].color" type="string"}
  Variant color (e.g., `Black`).
  ::
::
